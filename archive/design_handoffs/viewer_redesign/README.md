# Handoff: Viewer page redesign (RMC Phonon Dynamics)

## Overview
This package redesigns the **ViewerPage** of the RMC Phonon Dynamics web app ‚Äî the
results view a user lands on after running a phonon calculation (or loading a
saved `band.yaml`/`.json`). It has two tabs:

- **Bands + Mode** ‚Äî phonon band structure (E vs k) on the left, a 3D mode
  displacement viewer on the right, a mode-inspector readout, and a full set of
  3D rendering controls.
- **S(Q,E)** ‚Äî simulated inelastic-neutron-scattering heatmap + phonon DOS.

It is the sibling of the already-delivered Runner redesign
(`design_handoff_runner_redesign/`) and shares the exact same visual system.

## About the Design Files
The files in this bundle (`ConsoleViewer.dc.html`, `Viewer.dc.html`,
`support.js`) are **design references created in HTML** ‚Äî a prototype showing
the intended look and behavior. They are **not production code to copy
directly**. `support.js` is the prototype's own runtime and is irrelevant to the
target app.

The task is to **reskin the existing `web/src/pages/ViewerPage.jsx` and its
child components** in the real React codebase so they match this prototype, while
keeping the documented prop/data contract unchanged. **Do not modify the
compute/io/math layers** ‚Äî see "The boundary" in
`web/DESIGN_NOTES.md` (the project's existing spec). When the new UI feeds the
same props/shapes, everything keeps working (`npm run validate` guards them).

The two genuinely bespoke pieces ‚Äî `CrystalViewer` (three.js canvas) and the
INS heatmap/DOS canvases ‚Äî are **not** regenerated by this prototype. Reskin the
*controls and chrome around them*; keep the canvases. In the prototype the 3D
viewer and heatmap are SVG/canvas mocks purely to show layout and styling.

## Fidelity
**High-fidelity (hifi).** Final colors, typography, spacing, and interactions.
Recreate pixel-perfectly using the codebase's existing React patterns. Exact
tokens are listed below. The prototype is built at a 1320px max content width;
it should be fluid/responsive in the real app (the grids already collapse ‚Äî see
Responsive notes).

## Interactions & Behavior
- **Tab switch** ‚Äî toggles Bands+Mode ‚Üî S(Q,E) (local state). S(Q,E) draws on
  mount/update.
- **Band pick** ‚Äî click on the plot ‚Üí nearest (k, band) ‚Üí updates `selK`/`selM`
  ‚Üí 3D viewer shows that eigenvector, inspector + energy readout update.
- **k/band number inputs** ‚Äî clamp to `[1, nK]` / `[1, nModes]`, 1-indexed in
  the UI, 0-indexed in state.
- **E-range min/max** ‚Äî rescale the band y-axis; **Reset** clears them.
- **THz toggle** ‚Äî converts every displayed energy meV‚ÜíTHz (√ó0.2418), unit
  label follows.
- **Play/Pause** ‚Äî runs/pauses the mode oscillation (`isPlaying`).
- **WebM record** ‚Äî toggles `recording` (button goes red, REC indicator shows);
  **GIF** fires a one-shot capture signal.
- **Camera X/Y/Z** ‚Äî snaps the camera down that axis; **Reset** (3D card)
  resets camera + supercell to 2√ó2√ó1.
- **Supercell / amplitude / speed / atom size / bond cutoff / style / bonds /
  shading / vectors / cell** ‚Äî all feed `CrystalViewer` props live; memoize
  array/object props (`supercell`, `elementColors`, `elementRadii`, `bondRules`)
  so the three.js canvas doesn't rebuild and reset the camera on unrelated
  re-renders.
- **Per-element color/radius** ‚Äî live; "reset" restores defaults.
- **Run INS** ‚Äî (re)computes and redraws the heatmap + DOS; colormap & log
  toggle redraw without recompute.
- **Exports** ‚Äî band.yaml / band.json (`writers.js`), VASP (`vaspexport.js`),
  S(Q,E) CSV. Load reads a file via `fromBandText`.

### Animations
- Logo orbits spin 9s linear infinite.
- Status dot "blip" 2s; REC dot pulse 1s.
- Mode oscillation: atoms translate ¬±(displacement) sinusoidally; period derived
  from `speed` (prototype: `(0.34 ‚àí speed)√ó7` s), magnitude from `amplitude`.
  In the real app this is the three.js animation loop, not CSS.
